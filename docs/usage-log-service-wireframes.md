# Usage Log Service Wireframes
Version: v1.0  
Document Type: Low-Fidelity Wireframe Spec  
Basis: no-account architecture, dataset-centric access

---

## 1. 문서 목적

본 문서는 무계정형 Usage Log Service의 핵심 화면들에 대한 와이어프레임을 정의합니다.  
회원가입/로그인 없이 `dataset_id`와 key 기반으로 접근하는 구조를 전제로 하며, MVP 범위에서 필요한 주요 사용자 흐름과 화면 구성을 정리합니다.

### 핵심 사용자 흐름
1. 랜딩 페이지 진입
2. 새 데이터셋 생성
3. `dataset_id`, `ingest_key`, `admin_key` 확인 및 복사
4. 기존 데이터셋 접근
5. 대시보드에서 조회 / 필터 / 상세 확인
6. JSON / Excel export
7. 키 재발급 및 데이터셋 설정 관리

---

## 3. 공통 레이아웃 원칙

### 공통 Header
- 서비스 로고
- 서비스명
- 상단 우측 도움말 / 문서 링크
- 계정 메뉴 없음

### 공통 스타일 방향
- shadcn/ui 기준의 단정한 B2B 관리형 UI
- 정보 밀도는 중간 이상
- 테이블 중심

- 파괴적 액션은 명확히 분리

### 공통 반응형 정책
- Desktop 우선 설계
- Tablet까지 대응
- Mobile은 조회 중심 최소 대응

## 4.1 랜딩 페이지

### 목적
- 서비스 개념 전달
- 빠른 진입 제공
- 신규 데이터셋 생성 / 기존 데이터셋 접근 유도

### 주요 사용자 액션
- 새 데이터셋 만들기
- 기존 데이터셋 열기

### 주요 컴포넌트
- Hero
- 서비스 설명 카드
- 3-step 사용 흐름
- CTA 버튼 2개

```text

| LOGO  Usage Log Service                                             Docs   Help |
+----------------------------------------------------------------------------------+
|                                                                                  |
|      [헤드라인]                                                                  |

|                                                                                  |
|      설명 문구:                                                                  |
|      - 회원가입 없이 사용 가능                                                   |

+----------------------------------------------------------------------------------+
```

### UX 메모
- 랜딩은 설명보다 행동 유도가 우선
- CTA 2개를 시각적으로 크게 배치
- 코드 예시를 짧게 노출해 “개발자용 서비스” 인상을 줌

## 4.2 데이터셋 생성 페이지

### 목적
- 새로운 데이터셋을 생성
- 데이터셋 메타 정보 입력

### 입력 항목
- Dataset Name (필수)
- Description (선택)
- Dim Alias 1~10 (선택)
- Metric Alias 1~3 (선택)
- Example Use Case (선택)

### 권장 UX

- 고급 설정은 Accordion으로 접기

### 입력 항목
- Dataset ID
- Admin Key

### Wireframe

```text

+----------------------------------------------------------------------------------+

|                                                                                  |
|  기존 데이터셋 열기                                                              |
|  설명: dataset_id와 admin_key를 입력해 조회 및 export를 할 수 있습니다.         |

|  Dataset ID *                                                                    |
|  +--------------------------------------------------------------------------+    |
|  | ds_29XQ8A1                                                                |   |
|  +--------------------------------------------------------------------------+    |
|                                                                                  |
|  Admin Key *                                                                     |
|  +--------------------------------------------------------------------------+    |
|  | adm_xxxxxxxx_xxxxxxxxxxxxxxxxxxxxxxxxx                                    |   |
|  +--------------------------------------------------------------------------+    |
|                                                                                  |
```

- dataset_id 없음

- 접근 횟수 제한 초과

- 행 클릭 시 상세 Drawer 열기

### UX 메모
- 가장 중요한 페이지이므로 밀도와 가독성의 균형이 중요
- KPI는 4개 정도로 제한
- 필터 상태는 URL query param으로 반영하면 공유 가능성이 좋아짐

## 4.7 Export Modal

### 목적
- JSON / Excel 내보내기 옵션 설정
- 현재 필터 기준 export 여부 선택

### 주요 옵션
- Format: JSON / Excel
- Range: current filters / all data
- Include payload: on/off
- Visible columns only: on/off

### Wireframe

```text
+------------------------------------------------------------------+
| Export Data                                                   X  |
|------------------------------------------------------------------|
| Format                                                           |
| ( ) JSON                                                         |
| (•) Excel (.xlsx)                                                |
|                                                                  |
| Range                                                            |
| (•) Current filtered rows                                        |
| ( ) All rows                                                     |
|                                                                  |
| Options                                                          |
| [x] Visible columns only                                         |
| [x] Include dim / metric aliases                                 |
| [ ] Include payload_json sheet                                   |
|                                                                  |
| 예상 결과                                                        |
| - 약 1,842 rows                                                  |
| - columns: 11                                                    |
|                                                                  |
| [취소]                                          [Export 시작]    |
+------------------------------------------------------------------+
```

### UX 메모
- 필터 기준 export가 기본값
- 큰 데이터셋이면 row 수 안내 필요
- 장기적으로는 비동기 export job으로 확장 가능

# 9. 디자인 메모

- 정보 구조는 “개념 설명”보다 “즉시 실행”을 우선
- 이 서비스의 정체성은 대시보드보다 “간단한 로그 박스 + 내보내기”에 가깝다
- 화면 수를 늘리기보다 한 화면 안에서 작업이 끝나게 설계
- 무계정형 구조이므로 소유 개념보다 키 보관과 재발급 UX가 중요
- 대시보드의 중심은 차트가 아니라 테이블

# 11. 최종 정리

본 와이어프레임은 다음 철학을 따릅니다.

- 로그인 없는 무계정형 서비스
- dataset 중심 접근
- 테이블 중심 UI
- Excel 활용성을 고려한 dim / metric 구조
- payload는 상세 원본 보존용
- 빠른 생성, 빠른 적재, 빠른 export

이 문서를 기반으로 다음 단계에서는 실제 UI 설계안, 컴포넌트 명세, 페이지별 상태 전이도, 프론트엔드 구현 태스크로 확장할 수 있습니다.
